Week 2 Technical: Display Inline vs. Block vs. Inline Block

July 27th, 2014

When learning web design, it seems like there are a few things that beginners consistently get tripped up on. Absolute vs. relative positioning, class vs. id, and div vs. span have all caused me numerous headaches and frantic Google searches. This blog post is an attempt to explain the difference between displaying elements as inline, block, and inline-block.

div vs. span

First, a short explanation of the difference between a div and a span. Divs are considered "block" elements - they create a two-dimensional rectangle of space, and can be given height, width, margins, and padding. Spans, on the other hand, are "inline" elements - they don't create a rectangular block, and cannot take on properties like height, width, margins, or padding. Divs can contain other elements, and can only be contained by other block-level elements. Spans cannot contain block-level elements, and can be contained inside any type of element.

display: block;

By default, divs display as blocks. They take up as much space as you tell them to, and each new div appears on the next line, below the one above it. Giving a div the property display: block; in CSS is redundant - that's the way it will behave already. However, giving a span the display: block; property asks the span (normally inline) to behave like a div - as a block element.

display: inline;

On the flip side, the display: inline property is a way to tell a div to behave like a span. Rather than taking up the space that the rectangular div would normally carve out, the div will appear "inline" with the other elements it is near, and it will not automatically create any new lines below it.

display: inline-block;

Finally, you have display: inline-block; which gives you the best of both worlds! Telling a div to display in this way retains its "blockiness," letting it take up rectangular space, and padding and margins, but allows it to sit next to other inline elements. This is great for navigation bars, for example, where you may want each element to sit next to one another horizontally across the bar.
