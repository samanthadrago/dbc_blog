Week 8 Technical: Sorting Algorithms

September 5, 2014

Here's a shuffled deck of 52 cards. Sort it.

How would you approach this problem? Maybe you'd look at the first card in the deck, compare it to the second card, and move the second card to the top of the deck if it was smaller. Then, you'd repeat that with each new card, slotting it in between the cards that had already been sorted. It might take you around 3 minutes to sort the cards - not too bad. This is what's called an insertion sort, and it works great for small datasets like this one.

But what if instead of 52 cards, you were a librarian with 5,000 books to alphabetize? Insertion sort isn't looking too helpful anymore. On top of it being inefficient, you might run into physical limitations - you're going to exhaust yourself walking back and forth along your list of sorted books to slot them each in to place! What if you first divided the books into piles - a pile for each letter of the alphabet - and then sorted each pile individually using insertion sort? Then at the end, you'd combine your piles and voila! This would be much more efficient than trying to use insertion for the whole set of books. You're now using bucket sort, a "divide and conquer" style of sorting that breaks the (huge!) problem up into smaller, easier chunks.

Sorting is a quintessential programming problem. It is so fundamental to computer science that sorting questions commonly come up in technical job interviews. Even President Obama knows a thing or two about sorting! Check out this interesting animation that visualizes how different algorithms sort large datasets.

There are dozens of sorting algorithms out there, some of which are better than others for particular scenarios. Let's take a look at a few common sorting algorithms:

Insertion Sort

This is one of the simplest sorting algorithms; as you saw above, it's also one that is commonly used for "real world" sorting. Insertion sort iterates over each element in the list, removing the element and placing it in the sorted list in its appropriate position. This sort method is very inefficient on large datasets, but it is simple to implement and thus popular for smaller lists.

Bubble Sort

Another simple sort, bubble sort works by comparing adjacent values and switching their order if it is incorrect. It works through the list of values as many times as is needed for the whole list to become sorted. Bubble sort is a fairly inefficient algorithm mainly used to illustrate sorting concepts in computer science classes.

Merge Sort

Merge sort is one of the "efficient" sorts, meaning that it works relatively quickly on a large dataset. It works by dividing the dataset into sublists, then merging the sublists to create larger, sorted sublists, until the whole list is sorted. Merge sort is a popular default sorting algorithm for some languages, like Perl, Python (uses a variation called timsort), and Java.

Quicksort

Quicksort is also considered an efficient sort. It is similar to bucket sort in that it is a "divide and conquer" algorithm. A "pivot" point is selected; all elements larger than the pivot are placed after it, and all elements smaller than the pivot are placed before it. These sublists are again sorted with new pivots, until all the sublists have been sorted.

Computer scientists like coming up with "joke" sorting algorithms with names like "stooge sort" and "bogosort," which are extremely inefficient algorithms. For example, bogosort works by checking to see if the data is in order, and if it isn't, randomizing the order of the data and checking again, until the list happens to end up in order. This is a similar methodology to putting monkeys in a room with a typewriter and hoping for Shakespeare. While these types of sort algorithms serve no practical use, they are useful to contrast with efficient sorting algorithms.
